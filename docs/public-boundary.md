# Public Boundary

This repository is a public-safe portfolio edition of the live Fixera website system.

Fixera is a public-facing business, so the brand name and production website may appear as verification context. That does not make the operational system public. This repository keeps the public website story while excluding private business data, private automation, analytics exports, lead records, customer/job details, secrets, deployment state, and internal planning material.

## Public-Safe

- Source HTML, CSS, and JS for already public-facing pages
- Public Fixera brand context and production website reference
- Build and validation scripts
- Clean URL, sitemap, hreflang, and SEO architecture documentation
- High-level tracking boundary notes
- Case-study material
- Placeholder visual media that preserves expected file paths

## Excluded

- `.env*`
- `.vercel/`
- generated `public/`
- `reports/`
- `artifacts/`
- `audits/`
- `docs/research/`
- `docs/active/`
- Search Console exports
- GA4 exports
- lead reports
- growth reports
- operations exports
- active outreach strategy
- active business backlog
- private customer, job, lead, or revenue records
- real customer, property, job, or workspace photos
- live lead capture endpoints, private form IDs, webhooks, tokens, and account identifiers

## Rule

Public material may explain the architecture, validation pattern, and public brand context. It must not expose live operational data, private business decision logs, customer/job records, real property imagery, or private third-party account configuration.

Visual media in this edition is intentionally synthetic placeholder media. The repository keeps the production-like asset structure for engineering review while avoiding publication of real-world job imagery.
